Heart Disease Prediction Using Machine Learning

Clinical risk screening • Model comparison • Deployment-ready pipeline

Abstract

Concise summary

This project builds an end-to-end ML pipeline to predict heart disease from clinical features (age, BP, cholesterol, ECG metrics, and more). Models were trained, evaluated, compared and prepared for deployment as a web API for screening support.

Problem Statement & Scope

Why this matters

  • Problem: Early detection of heart disease is critical — manual screening is time-consuming and variable.
  • Scope: Use routinely available clinical parameters to build a reproducible ML model for screening and clinician-assisted decision support.

Objectives

  • Train and compare multiple classifiers on a hold-out test set.
  • Provide interpretability guidance (feature importance / SHAP).
  • Prepare a deployment-ready pipeline (Flask API + saved model).

Methodology & Workflow

Clear step-by-step process (vertical)

1. Data Collection

Gather dataset from reliable sources (UCI / Kaggle or hospital records). Load CSV/Excel into a DataFrame and inspect for schema, missing values and basic stats.

2. Data Visualization

Perform EDA: histograms, boxplots, correlation heatmap and scatterplots to detect distribution, outliers, and relationships between features and target.

3. Splitting the Features and Target

Separate the dataset into X (features) and y (target). Drop irrelevant columns and create any derived features needed for modeling.

4. Train — Test Split

Split data into training and testing sets (common splits: 80/20 or 75/25). Use stratified sampling for imbalanced classes to preserve class ratio.

5. Model Training

Train selected models (Logistic Regression, Naive Bayes, KNN, Decision Tree, SVM) with consistent preprocessing pipelines. Use cross-validation (StratifiedKFold) and tune hyperparameters.

6. Model Evaluation

Evaluate models on the test set using Accuracy, Precision, Recall, F1-score, and ROC-AUC. Inspect confusion matrices and compare model trade-offs based on the use-case (favor recall for screening).

7. Predicting Results

Use the persisted pipeline (scaler/encoder + model) to predict on new patient inputs. Return class label and probability/confidence; provide explanation (top contributing features) when possible.

Deployment & Tools

Suggested stack & notes

  • Languages & Libraries: Python, scikit-learn, pandas, numpy, matplotlib, joblib.
  • Web framework: Flask (use `app_flask.py` to serve API).
  • Persist pipeline: Save preprocessing + model with joblib (`joblib.dump(pipeline, 'saved_models/model.pkl')`).
  • Production: Containerize with Docker, run under Gunicorn behind Nginx, enable HTTPS and logging/monitoring.
